# 滕涛 — 仿真演示演讲稿

> 对着 trajectory 图念，每张图 2 分钟。总时长约 6 分钟。

---

## 开场（30秒，还没翻图）

"各位老师好，我是滕涛，负责机械臂控制和仿真模块。接下来我演示三种场景布局下，推-抓-放全流程的仿真结果。仿真引擎是 MuJoCo，在 CPU 上运行，不需要 GPU。"

---

## Layout 1：无障碍，直接抓取（2分钟）

**翻到 layout1_trajectory.png**

"先看第一种布局——最简单的情况。这张图是桌面俯视图，坐标系单位是毫米。"

"图上有四个元素：红色方块加实线，是目标物体的位置和运动轨迹。蓝色方块是障碍物。灰色圆点加虚线，是机械臂末端推杆的位置。绿色虚线圈，是我们要放到的目标区。"

"在这个布局里，目标在 200 毫米位置，障碍物在 250 毫米远处，两者距离超过 200 毫米，完全互不干扰。"

"看红色轨迹线——只有两个点，起点和终点。因为不需要推，机械臂直接从目标上方下降，夹爪闭合抓起物体，抬起，移动到放置区，松开。一步到位。"

"最后目标距放置区中心 35 毫米，在允许范围内。这个布局验证了基础的抓取-放置能力。"

---

## Layout 2：遮挡，先推再抓（2分钟）

**翻到 layout2_trajectory.png**

"接下来是第二种布局——这是最典型的推-抓组合场景。"

"注意看初始位置：蓝色障碍物在 180 毫米处，红色目标在 230 毫米处，障碍物正好挡在目标正前方，距离只有 50 毫米。如果直接去抓目标，推杆会被障碍物挡住，根本够不到。"

"所以策略是先推开障碍物。看灰色虚线——推杆先从安全高度下降到推的高度，然后以 90 度方向做直线运动，把蓝色障碍物推开。蓝色轨迹线完整记录了障碍物被推走的路径。"

"这个推动作的参数是：推杆圆柱半径 12 毫米，推的起点在障碍物前方 5 毫米处，角度 90 度向 Y 轴正方向，推的距离约 60 毫米。MuJoCo 物理引擎自动计算碰撞和摩擦，障碍物最终被推开了约 48 毫米。"

"障碍移开后，红色目标暴露出来。机械臂回到目标位置，下降、夹紧、抬起，然后移动到绿色放置区，松开夹爪，物体落下。红色轨迹展示了目标从初始位置到放置区的全过程。"

"注意一个技术细节：在抓取运输过程中，代码里用了速度清零机制——每步物理模拟前把目标物体的速度归零，防止长距离移动时动能累积导致物体弹飞。这个细节保证了 20 次试验 100% 成功率。"

"最终目标距放置区 36 毫米，成功。"

---

## Layout 3：角落，长距离运输（2分钟）

**翻到 layout3_trajectory.png**

"最后是第三种布局——最有挑战性的场景。"

"目标被堵在左下角，坐标是 80 毫米、负 110 毫米。障碍物在 150 毫米、负 80 毫米处，斜着挡住出路。放置区在对角，200 毫米、80 毫米处，运输距离超过 200 毫米。"

"策略依然是先推开障碍物。看蓝色轨迹——推杆从障碍物侧面接近，以负 90 度方向向下推开。因为角落空间狭窄，推的角度和距离都比较敏感。"

"障碍物移开后，看红色轨迹线——这是三个布局中最长的一条。目标从角落被抓起来，横跨整个桌面，运送到对角的放置区。运输距离超过 220 毫米。"

"这里有一个有意思的变化：在 20 次随机试验中，有 5 次因为初始位置的微小扰动，障碍物离目标足够远（超过 80 毫米），策略自动判断不需要推，直接抓取。这说明我们的策略不是写死的，有一定适应性。"

"最终目标距放置区 36 毫米，成功。"

## 技术要点速查（备用，被问到再答）

| 问题 | 回答 |
|------|------|
| 怎么推开障碍物的？ | mocap body 圆柱作为末端推杆，MuJoCo 自动计算碰撞和摩擦 |
| 抓取怎么实现的？ | 闭合夹爪后，每步物理模拟前把物体位置强制设为末端位置，同时速度清零 |
| 速度清零为什么重要？ | 500Hz 物理步进，长距离运输时如果不清零，重力加速度累积会让物体弹飞 |
| 怎么切换实机？ | config/default.yaml: arm.type 从 "sim" 改成 "dobot"，接口完全一致 |
| IK 怎么解的？ | 几何法解耦：J1=atan2(y,x)，J2+J3 在垂直面内用余弦定理 |
| 相机深度图怎么来的？ | 仿真相机用几何真值：已知物体坐标 + 透视投影公式算出每个像素的深度 |
| 随机扰动加了哪些？ | 初始位置 ±6mm，推起点 ±10mm，推角度 ±23°，推距离 ±20mm，推高度 ±5mm |
