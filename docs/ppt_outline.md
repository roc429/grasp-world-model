## 第 2 页 · 问题背景

```
标题: 当物体被遮挡，机械臂怎么办？

┌──────────────┐    ┌──────────────┐    ┌──────────────┐
│  🧱 障碍物    │    │  📦 目标物体   │    │  🎯 放置区    │
│ 挡住目标     │ → │ 无法直接抓取  │ → │ 需要先推     │
│              │    │              │    │ 后才能抓     │
└──────────────┘    └──────────────┘    └──────────────┘

核心挑战:
  ① 如何预测"推"这个动作会产生什么效果？
  ② 如何在无数种可能的推法中选出最优的那一个？
  ③ 预测不准怎么办？

传统方法: 规则式 if-else → 无法适应新场景
我们的方案: 世界模型 + 前瞻规划 → 想象→规划→执行 闭环
```

## 第 6 页 · 世界模型（一）— 设计

```
标题: 世界模型: 让机器"想象"动作的后果

核心问题: 给定 (当前状态 s, 候选动作 a), 预测下一状态 s'

┌────────────────────────────────────────────┐
│  模型结构: MLP 残差网络                     │
│                                            │
│  s(10) + a(4) = 14                         │
│        ↓                                   │
│  Linear(14, 256) + ReLU                    │
│        ↓                                   │
│  Linear(256, 256) + ReLU                   │
│        ↓                                   │
│  Linear(256, 10) → Δs                      │
│        ↓                                   │
│  s' = s + Δs  (残差连接)                    │
│                                            │
│  参数量: ~69K | 训练设备: DCU 服务器         │
└────────────────────────────────────────────┘

为什么用残差？预测变化量 Δs 比预测绝对位置更容易学习
```

## 第 8 页 · 前瞻规划

```
标题: CEM 规划器: 在动作空间中搜索最优

┌─────────────────────────────────────────────┐
│  Cross-Entropy Method (CEM)                 │
│                                             │
│  初始化: 动作分布 N(μ₀, σ₀²)                 │
│                                             │
│  for iteration = 1..K:                      │
│    ① 从当前分布采样 200 条候选动作            │
│    ② 用世界模型预测每条动作的效果 + 评分       │
│    ③ 选 top 20% 精英 (40条)                 │
│    ④ 更新分布 μ = mean(精英), σ = std(精英)  │
│                                             │
│  输出: 评分最高的动作                         │
│                                             │
│  参数: K=5, N=200, elite_ratio=0.2          │
└─────────────────────────────────────────────┘

评分函数 = graspable_weight × 可抓取性 − distance_weight × 距离惩罚 − collision_penalty × 碰撞
```

## 附录: 可能被问到的技术问题

| 问题 | 回答要点 |
|------|---------|
| 为什么用 CEM 不用 MCTS？ | 连续动作空间, CEM 天然适合; MCTS 适合离散空间 |
| 世界模型为什么不直接用物理仿真？ | 物理仿真太慢(秒级), 神经网络推理快(毫秒级) |
| 为什么选残差结构？ | 预测 Δs 比绝对 s 更容易; 物体静止时 Δs=0, 模型输出天然趋零 |
| 5000 条数据够吗？ | 任务状态空间不大(桌面2D), 5000条覆盖了主要动作分布 |
| 怎么保证实机安全？ | 软件安全范围检查 + 硬件急停 + 低速度参数 |
| 和 DreamerV3 什么关系？ | 借鉴 RSSM 思路, 但针对桌面推-抓做了简化: 确定性 MLP 替代随机隐变量 |
